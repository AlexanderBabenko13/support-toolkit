\# AI Workflow для SupportToolkit



\## Роль AI



AI помогает писать, улучшать и проверять PowerShell-код, но не должен предлагать опасные изменения без явного подтверждения.



\## Основные правила



1\. Не удалять файлы без подтверждения.

2\. Не менять системные настройки без подтверждения.

3\. Не использовать сторонние модули без причины.

4\. Все функции должны иметь понятные названия.

5\. Все ошибки должны обрабатываться через try/catch.

6\. Код должен быть читаемым для L2 инженера.

7\. Комментарии и сообщения пользователю писать на русском языке.



\## Формат задач для Cursor



Сначала:

\- объясни план;

\- покажи, какие файлы будешь менять;

\- только потом предлагай код.



\## Для PowerShell



Использовать:

\- Set-StrictMode;

\- функции;

\- понятные имена переменных;

\- безопасные проверки;

\- логирование;

\- режим DryRun для потенциально изменяющих действий.



Не использовать:

\- скрытые изменения системы;

\- автозапуск;

\- обход политик безопасности;

\- скачивание внешних файлов без необходимости.
